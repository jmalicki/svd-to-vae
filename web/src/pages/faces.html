Squeezing a face through a few numbers

Source on GitHub

So far the SVD lived on toy matrices. This page is a real application of the same math: compress actual face photos with truncated SVD, and watch whether a handful of numbers still look like this person.

From truncated SVD to a face

Why bother? A face is thousands of pixels. If the truncated SVD from the previous page still works when the matrix entries are pixels, then you can store or send a person as a short list of numbers instead of a whole image — and you can see what that compression costs, because the rebuild is a picture. That is the same recipe as before (keep the top $k$ singular components, discard the rest), aimed at something you already care about recognizing.

Concretely: each lined-up photo becomes one long column of pixel values; about 240 columns make a matrix $X$. Its SVD is unchanged in meaning — shared directions ordered by $\sigma_1 \ge \sigma_2 \ge \cdots$, and for each face a list of coordinates along those directions. That list is the face’s representation. Truncating to rank $k$ keeps only the first $k$ coordinates and rebuilds $\text{average face} + \sum_{i=1}^{k} c_i\,u_i$. The demo is that reconstruction next to the original — so you can judge with your eyes when the math has kept “enough.”

Scrub $k$ to change how many singular components you keep. Full rank ($N-1$) can match the left tile; small $k$ is the interesting squeeze. (We line faces up before the SVD so compression isn’t wasted on “where is the eye?” — warp details in the appendix.)

Singular directions for this face

Truncated SVD keeps the top $k$ pieces ordered by singular value $\sigma_1 \ge \sigma_2 \ge \cdots$ (largest stretch first — same idea as on the truncate page). Each bar is this face’s coordinate along that shared direction. The amber curve is the singular values themselves (shared by everyone). Blue = kept at this $k$; gray = thrown away.

Appendix — names for the curious

  • What is stored for each face. After centering (subtract the average face), column $j$ of $X$ has coordinates $c_{1j},c_{2j},\ldots$ along the left singular vectors $u_1,u_2,\ldots$ of the stack. Those $c$’s are the bars below; the amber curve is $\sigma_1,\sigma_2,\ldots$. Rank-$k$ rebuild is $\hat x_j = \bar x + \sum_{i=1}^{k} c_{ij}\,u_i$ (ml-matrix SVD).
  • Warp / alignment. Raw photos put eyes and mouths in different places; a pixel-wise SVD would waste early singular components on “where is the feature?” We mark the same landmarks on every face, Procrustes-align shapes, then piecewise-affine warp on a Delaunay mesh (delaunator) onto a shared mean shape. The warp does not reduce dimension — it only puts correspondence into the columns before the SVD.
  • Demo panels. Left = lined-up original column. Middle = truncated-SVD rebuild at the current $k$. Right = $k=0$: nothing kept, just the average face.
  • Data. IMM Face Database — 240 landmarked faces (40 subjects × 6). Cite: M. B. Stegmann, B. K. Ersbøll, and R. Larsen. FAME – a flexible appearance modelling environment. IEEE Trans. on Medical Imaging, 22(10):1319–1331, 2003.
