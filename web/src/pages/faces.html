Squeezing a face through a few numbers

Source on GitHub

On the previous page, a truncated SVD compressed a matrix by keeping only the top $k$ pieces. Here that matrix is literal: each lined-up face is one long column of pixel values, and we stack about 240 such columns into one big matrix $X$. The truncated SVD of $X$ finds $k$ shared “face directions” plus, for every person, $k$ numbers that say how much of each direction to mix in. Rebuild those $k$ numbers and you get an approximate picture — same math as before, but now the approximation is a face you can recognize (or fail to).

From truncated SVD to a face

  1. Same truncation, different matrix. Before, the entries of $A$ were abstract numbers. Now each column of $X$ is one face (thousands of pixels). Truncated SVD still means: keep only rank $k$, throw the rest away, rebuild. Each face becomes $k$ coefficients plus a shared decoder made from those top pieces. Small $k$ = aggressive compression (more generic). Large $k$ = closer to the original.

  2. Line the faces up first (so compression isn’t wasted on “where is the eye?”). Raw photos have features in different places; compressing them as-is smears an eye onto a cheek. We mark the same spots on every face and gently stretch each photo so those spots land on the same places — the “warp.” Still a full-resolution picture, just lined up. Only the SVD step that follows is the actual compression.

  3. Rebuild means paint a picture from those $k$ numbers. Left = lined-up original. Middle = truncated-SVD reconstruction (rank $k$). Right = $k=0$: nothing kept, just the average face — maximum compression, no identity.

Drag $k$ below: you are changing the truncation rank. The slider goes up to full rank ($N-1$ for $N$ faces) so the middle can match the left; small $k$ is the interesting compression. Names like Procrustes and Delaunay are in the appendix.

Appendix — names for the curious

  • Warp / alignment. We use landmark Procrustes alignment, then a piecewise-affine warp on a Delaunay mesh (delaunator) so every face shares one mean shape. The warp does not reduce dimension; it only changes coordinates.
  • Bottleneck / rebuild. Same story as truncated SVD on the previous page: keep rank $k$, discard the rest, decode. Here the matrix columns are lined-up face images; the middle panel is $\hat x$ from those $k$ coefficients (ml-matrix SVD).
  • Data. IMM Face Database — 240 landmarked faces (40 subjects × 6). Cite: M. B. Stegmann, B. K. Ersbøll, and R. Larsen. FAME – a flexible appearance modelling environment. IEEE Trans. on Medical Imaging, 22(10):1319–1331, 2003.
