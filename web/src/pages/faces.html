Squeezing a face through a few numbers

Source on GitHub

Time to connect the math to something in the real world. Up to now the SVD acted on toy matrices; here we point the same truncated SVD at a stack of face photos and ask a practical question: can a handful of numbers still look like this person?

From truncated SVD to a face

Faces are a good real-world test for two reasons. First, you already know what “still looks like this person” means — you do not need a formula to judge the result. Second, a photo is thousands of pixels, so if we can store someone as a short list of numbers instead of the whole image, that is genuinely useful.

  1. Start from what you already know. On the previous page we took a matrix, wrote it with an SVD, and then truncated it: keep only the top $k$ singular components (the ones with the largest singular values), throw the rest away, and rebuild an approximation. That recipe never cared what the numbers in the matrix meant. They could have been anything.

  2. Now the numbers are pixels. Take each lined-up face photo and stretch it into one long column of pixel values. Stack about 240 such columns side by side. Call that big matrix $X$. We are about to do the same truncated SVD as before — only now every entry is a brightness in a face.

  3. What the SVD gives you. Same objects as always: a list of shared directions for the whole stack, ordered by singular value (most important first), and — for each face — a list of coordinates along those directions. Think of the shared directions as reusable “face patterns,” and the coordinates as how much of each pattern this person needs.

  4. That coordinate list is the face’s representation. Instead of keeping thousands of pixels, we keep those coordinates. Truncating to rank $k$ means keeping only the first $k$ of them (the ones tied to the largest singular values) and rebuilding a picture from just that short list. The middle tile in the demo is exactly that rebuild, sitting next to the original so you can see what was lost.

Scrub $k$ below. Large $k$ can match the left tile; small $k$ is the interesting squeeze — when does the rebuild stop looking like this person? (We line faces up before the SVD so early components aren’t wasted on “where is the eye?” — warp details and the rebuild formula live in the appendix.)

Singular directions for this face

Truncated SVD keeps the top $k$ pieces ordered by singular value $\sigma_1 \ge \sigma_2 \ge \cdots$ (largest stretch first — same idea as on the truncate page). Each bar is this face’s coordinate along that shared direction. The amber curve is the singular values themselves (shared by everyone). Blue = kept at this $k$; gray = thrown away.

Appendix — names for the curious

  • What is stored for each face. After centering (subtract the average face), column $j$ of $X$ has coordinates $c_{1j},c_{2j},\ldots$ along the left singular vectors $u_1,u_2,\ldots$ of the stack. Those $c$’s are the bars below; the amber curve is $\sigma_1,\sigma_2,\ldots$. Rank-$k$ rebuild is $\hat x_j = \bar x + \sum_{i=1}^{k} c_{ij}\,u_i$ (ml-matrix SVD).
  • Warp / alignment. Raw photos put eyes and mouths in different places; a pixel-wise SVD would waste early singular components on “where is the feature?” We mark the same landmarks on every face, Procrustes-align shapes, then piecewise-affine warp on a Delaunay mesh (delaunator) onto a shared mean shape. The warp does not reduce dimension — it only puts correspondence into the columns before the SVD.
  • Demo panels. Left = lined-up original column. Middle = truncated-SVD rebuild at the current $k$. Right = $k=0$: nothing kept, just the average face.
  • Data. IMM Face Database — 240 landmarked faces (40 subjects × 6). Cite: M. B. Stegmann, B. K. Ersbøll, and R. Larsen. FAME – a flexible appearance modelling environment. IEEE Trans. on Medical Imaging, 22(10):1319–1331, 2003.
